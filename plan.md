# plan.md
## Поэтапный план реализации Debet

Документ основан на [PROJECT.md](./PROJECT.md).  
Текущее состояние: минимальный Expo-проект с главным экраном и заглушками кнопок.

---

## Этап 0. Подготовка инфраструктуры

**Цель:** привести проект к целевому стеку из PROJECT.md.

1. Установить зависимости:
   - `expo-router` — файловая навигация
   - `react-native-paper` + `react-native-safe-area-context` — UI-компоненты
   - `expo-sqlite` — локальная база данных
   - `expo-file-system` — хранение фотографий
   - `expo-image-picker` — выбор и съёмка фото
   - `expo-location` — GPS-координаты
   - `react-native-maps` (или `expo-maps`) — отображение проекта на карте
2. Перевести точку входа на Expo Router (`app/_layout.tsx`, `app/index.tsx`).
3. Настроить общую тему React Native Paper (цвета, типографика).
4. Создать каркас папок: `app/`, `components/`, `db/`, `repositories/`, `types/`, `utils/`.
5. Удалить или заменить текущий `App.tsx` после миграции на Router.

**Результат:** приложение запускается, есть пустые экраны и единый стиль.

---

## Этап 1. Слой данных (SQLite)

**Цель:** локальное хранение без интернета, связь Projects → Budget → Expenses.

1. Спроектировать схему БД:
   - **projects** — поля из PROJECT.md (`id`, `name`, `description`, `visitlater`, `liked`, `dd`, `createdAt`)
   - **budget_items** — пункты бюджета проекта (`id`, `projectId`, `name`, `plannedAmount`, `order`, `createdAt`)
   - **expenses** — расходы (`id`, `projectId`, `budgetItemId`, `amount`, `description`, `createdAt`)
   - **executors** — исполнители (`id`, `name`, `phone`, `email`, `note`)
   - **project_executors** — связь проект ↔ исполнитель (`projectId`, `executorId`)
   - **photos** — метаданные фото (`id`, `projectId`, `filePath`, `createdAt`)
2. Написать модуль инициализации БД и миграций (`db/schema.ts`, `db/migrations.ts`).
3. Реализовать репозитории с CRUD-операциями для каждой сущности.
4. Добавить утилиты для работы с файлами фотографий (копирование в каталог приложения, удаление при удалении записи).
5. Написать вспомогательные функции:
   - сумма расходов по проекту / по пункту бюджета;
   - остаток бюджета и признак превышения;
   - определение «следующего пункта бюджета» (первый незакрытый по порядку).

**Результат:** данные сохраняются локально, связи между сущностями работают.

---

## Этап 2. Навигация и главный экран

**Цель:** реализовать структуру экранов из раздела 5 PROJECT.md.

1. Главный экран (`app/index.tsx`):
   - заголовок «Debet»;
   - пять кнопок: Создать проект, Создать расход, Проекты, Отчёты, Настройки.
2. Зарегистрировать маршруты:
   - `app/project/create.tsx` — создать проект
   - `app/project/[id].tsx` — детальная карточка проекта
   - `app/expense/create.tsx` — создать расход
   - `app/expense/[id].tsx` — детальная карточка расхода
   - `app/projects/index.tsx` — список проектов
   - `app/reports/index.tsx` — отчёты
   - `app/reports/form.tsx` — форма отчёта
   - `app/settings/index.tsx` — настройки
3. Добавить общий layout с Appbar (кнопка «Назад» на вложенных экранах).
4. Проверить переходы со всех кнопок главного экрана.

**Результат:** пользователь может перемещаться между всеми экранами.

---

## Этап 3. Режим «Проекты»

**Цель:** полный CRUD проектов согласно разделу 4.1 PROJECT.md.

1. **Создать проект** (`project/create.tsx`):
   - форма: название, описание, флаги `visitlater` и `liked`;
   - опционально: получение GPS-координат (`dd`) через `expo-location`;
   - прикрепление фотографий через `expo-image-picker`;
   - сохранение в SQLite и файловую систему.
2. **Список проектов** (`projects/index.tsx`):
   - отображение всех проектов (карточки React Native Paper);
   - фильтры: «не закончен», «понравилось»;
   - переход в детальную карточку.
3. **Детальная карточка проекта** (`project/[id].tsx`):
   - просмотр и редактирование полей;
   - галерея фотографий;
   - блок бюджета (пункты и фактические расходы);
   - список исполнителей с контактами;
   - индикатор превышения бюджета;
   - подсказка «следующий пункт бюджета»;
   - кнопка «Открыть на карте» (если заданы координаты `dd`).
4. **Карта** — экран или модальное окно с маркером проекта по `dd`.

**Результат:** пользователь создаёт, просматривает и редактирует проекты офлайн.

---

## Этап 4. Бюджет проекта

**Цель:** фиксация пунктов бюджета и анализ по мере выполнения проекта.

1. На экране проекта — раздел «Бюджет»:
   - добавление / редактирование / удаление пунктов (`name`, `plannedAmount`, порядок);
   - отображение плана, факта и остатка по каждому пункту.
2. Визуальная сигнализация:
   - превышение бюджета проекта в целом;
   - превышение по отдельному пункту.
3. Логика «следующий пункт бюджета»:
   - пункт с наименьшим `order`, у которого факт < плана (или не привязаны все расходы).

**Результат:** бюджет проекта ведётся и анализируется в реальном времени.

---

## Этап 5. Расходы

**Цель:** учёт расходов в разрезе проектов и пунктов бюджета.

1. **Создать расход** (`expense/create.tsx`):
   - выбор проекта;
   - выбор пункта бюджета (из выбранного проекта);
   - сумма, описание, дата;
   - сохранение в SQLite.
2. **Детальная карточка расхода** (`expense/[id].tsx`):
   - просмотр и редактирование;
   - ссылка на проект и пункт бюджета.
3. На карточке проекта — список всех расходов с группировкой по пунктам бюджета.
4. После сохранения расхода — пересчёт остатков и обновление индикаторов превышения.

**Результат:** расходы привязаны к проектам и бюджету, суммы пересчитываются автоматически.

---

## Этап 6. Исполнители

**Цель:** привязка проекта к людям с контактами (раздел 2 PROJECT.md).

1. CRUD исполнителей: имя, телефон, email, заметка.
2. На карточке проекта — привязка / отвязка исполнителей.
3. Возможность выбрать существующего исполнителя или создать нового.
4. Быстрый вызов / отправка email через системные intent (опционально для MVP).

**Результат:** у каждого проекта есть список исполнителей с контактами.

---

## Этап 7. Отчёты

**Цель:** сводный анализ проектов (экраны «Отчёты» и «Форма отчёта»).

1. **Список отчётов** (`reports/index.tsx`):
   - выбор проекта(ов) для анализа;
   - фильтр по периоду (дата создания расходов).
2. **Форма отчёта** (`reports/form.tsx`):
   - сводка: план / факт / остаток по проекту;
   - разбивка по пунктам бюджета;
   - список расходов за период;
   - отметка проектов `visitlater` / `liked` для включения в отчёт.
3. Отображение ключевых выводов: превышения, следующий пункт бюджета, общий итог.

**Результат:** пользователь видит аналитику по проектам без выхода в интернет.

---

## Этап 8. Настройки

**Цель:** базовые параметры приложения.

1. Экран настроек (`settings/index.tsx`):
   - информация о версии приложения;
   - единица валюты (символ для отображения сумм);
   - управление разрешениями (камера, геолокация);
   - очистка всех данных (с подтверждением).
2. Сохранение настроек в AsyncStorage или отдельной таблице SQLite.

**Результат:** пользователь может настроить отображение и управлять данными.

---

## Этап 9. Доработка и стабилизация MVP

**Цель:** готовое к использованию офлайн-приложение.

1. Обработка ошибок: пустые списки, отсутствие координат, отказ в разрешениях.
2. Валидация форм (обязательные поля, корректность сумм).
3. Единообразный UI на всех экранах (React Native Paper).
4. Тестирование на Android и iOS:
   - создание проекта с фото и координатами;
   - ввод бюджета и расходов;
   - проверка сигналов превышения;
   - формирование отчёта;
   - работа без интернета.
5. Обновить README с инструкцией по запуску и описанием функций.

**Результат:** стабильный MVP, соответствующий PROJECT.md.
